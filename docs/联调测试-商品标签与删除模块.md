# 联调测试：商品标签与删除模块

## 一、先做一次数据库迁移（旧库必做）

如果你的数据库是之前版本，请先执行：

```sql
-- 文件：db/migrate_add_product_tag.sql
ALTER TABLE t_product
    ADD COLUMN product_tag VARCHAR(32) NOT NULL DEFAULT '常规商品'
    COMMENT '商品标签：常规商品/当季限定/地区限定/自定义'
    AFTER category;
```

> 新建库直接跑 `schema.sql` 时已包含该字段，不需要重复执行。

## 二、页面功能

### 1）新增商品（带标签）

路径：收银点单页 -> `新增商品`

- 标签支持手动输入（例如：`当季限定`、`地区限定`、`樱花季限定`）
- 不输入时默认 `常规商品`

### 2）显示标签

收银点单商品卡片会显示标签：
- 无标签/空标签 -> 显示 `常规商品`

### 3）删除商品

商品卡片右上角“垃圾桶”按钮：
- 点击后会二次确认
- 删除后商品从在售列表移除
- 后端实际行为为**下架**（`status=0`），避免影响历史订单

## 三、接口说明

- `POST /api/product`：新增商品（支持 `product_tag`）
- `DELETE /api/product/{id}`：删除商品（逻辑下架）

# 联调测试：商品标签与删除

## 一、先执行数据库补丁（老库必须执行）

在 DataGrip / MySQL 执行：

`db/migrate_add_product_tag.sql`

作用：
- 给 `t_product` 增加 `product_tag`
- 默认值为 `常规商品`

## 二、功能点

- 新增商品时可输入“标签”
  - 可填：当季限定 / 地区限定 / 其他自定义
  - 不填默认：常规商品
- 已有老商品若没有标签，前端展示默认“常规商品”
- 商品删除（软删除）
  - 调用 `DELETE /api/product/{id}`
  - 实际将 `status` 置为 0，从在售列表移除

## 三、页面联调步骤

1. 进入“收银点单”
2. 点击“新增商品”
3. 录入名称、分类、售价、标签（可空）
4. 保存后在商品卡片可看到标签徽章
5. 点击商品卡片右上角垃圾桶删除，确认后商品从列表移除
