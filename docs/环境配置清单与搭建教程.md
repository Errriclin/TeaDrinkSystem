## 中小型茶饮店会员与进销存一体化管理系统

## 环境配置清单与搭建教程（不使用 Canal 版）

> 说明：本项目最终方案不再部署 Canal，改为由后端定时任务从 MySQL 同步热点数据到 Redis。本清单和教程以 Windows 10 环境为例，其他平台可类比。

---

## 一、整体环境概览

### 1.1 必选组件一览

- **JDK**：JDK 8（统一使用 8）
- **Maven**：3.6+（用于构建 Spring Boot 后端）
- **Node.js + npm**：Node 16+（建议 18），用于构建 Vue3 前端
- **MySQL**：5.7+ 或 8.x（项目主数据库）
- **Redis**：5.x+ 或 6.x（缓存，定时任务从 MySQL 刷新）
- **Nginx**：1.18+（前端静态资源托管 + 反向代理后端 API）
- **IDE**（可选但推荐）：
  - 后端：IntelliJ IDEA / Eclipse / VS Code + Java 插件
  - 前端：VS Code / WebStorm 等

### 1.2 目录规划建议

在 `d:\teadrinkSystem` 下推荐结构（已部分存在，可按需调整）：

- `docs/`：项目文档（需求、计划、本教程等）
- `backend/`：后端 Spring Boot 源码与配置
- `frontend/`：前端 Vue3 工程
- `db/`：SQL 脚本（建表、初始化数据、视图、触发器等）
- `deploy/`：Nginx 配置示例、启动脚本等

## 二、JDK 与 Maven 环境搭建

### 2.1 安装 JDK

1. 前往 Oracle 或 OpenJDK 官网下载 JDK 8。  
2. 安装到例如：`C:\Program Files\Java\jdk1.8.0_xx`（`xx` 为小版本号）。
3. 配置环境变量（以 Windows 为例）：
   - 右键「此电脑」→「属性」→「高级系统设置」→「环境变量」。
   - 新建系统变量 `JAVA_HOME`：  
     - 值：`C:\Program Files\Java\jdk1.8.0_xx`
   - 在 `Path` 中新增：`%JAVA_HOME%\bin`
4. 打开 PowerShell 或 CMD，输入：
   - `java -version`  
   - `javac -version`  
   若能正确输出版本号，说明 JDK 安装成功。

### 2.2 安装 Maven

1. 从 Maven 官方下载二进制包（例如 `apache-maven-3.8.8-bin.zip`），解压到如：`C:\dev\apache-maven-3.8.8`。
2. 配置环境变量：
   - 新建系统变量 `MAVEN_HOME`：  
     - 值：`C:\dev\apache-maven-3.8.8`
   - 在 `Path` 中新增：`%MAVEN_HOME%\bin`
3. 在 PowerShell/CMD 中执行：
   - `mvn -v`  
   若输出 Maven 版本与 Java 信息，则 Maven 配置成功。

## 三、Node.js 与前端开发环境

### 3.1 安装 Node.js

1. 前往 Node.js 官网下载 LTS 版本（推荐 v18+），直接下一步安装。  
2. 安装过程中勾选「Add to PATH」选项。
3. 打开终端检查版本：
   - `node -v`
   - `npm -v`

### 3.2 初始化前端项目（示意）

> 实际命令可稍后根据项目选择 Vite 配置，这里给出示意流程。

1. 在 `d:\teadrinkSystem` 目录下打开终端：  
   `cd d:\teadrinkSystem`
2. 使用 Vite 创建 Vue3 项目（示例）：
   - `npm create vite@latest frontend -- --template vue`
3. 进入前端目录安装依赖：
   - `cd frontend`
   - `npm install`
4. 安装 Element Plus：
   - `npm install element-plus`
5. 开发时启动本地服务：
   - `npm run dev`
6. 构建产物（将来用于 Nginx 部署）：
   - `npm run build`（生成 `dist/` 目录）

## 四、MySQL 数据库环境

### 4.1 安装 MySQL（Windows）

1. 可使用 MySQL 安装包或 Zip 版；推荐通过 MySQL Installer 安装，选择 MySQL Server 5.7 或 8.x。
2. 安装过程设置 root 密码（请记录），默认端口 3306。
3. 安装完成后，使用 MySQL Workbench / DBeaver / 命令行测试连接。

### 4.2 创建数据库与用户

1. 连接 MySQL 后，执行示例 SQL（可根据实际修改）：

```sql
CREATE DATABASE teadrink_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'teadrink'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON teadrink_system.* TO 'teadrink'@'%';
FLUSH PRIVILEGES;
```

2. 后续在 `db/` 目录下会提供建表与初始化脚本，可在此库中执行。

### 4.3 准备数据库脚本

- 在 `db/` 下计划存放：
  - `schema.sql`：建表、索引、约束
  - `views.sql`：视图创建 SQL（如销售统计视图）
  - `triggers.sql`：触发器 SQL（如库存预警触发器）
  - `init_data.sql`：必要的初始化数据（基础分类、角色等）

## 五、Redis 环境（用于缓存 + 定时同步）

### 5.1 安装 Redis（Windows 方案）

> 官方 Redis 未提供 Windows 版服务端，可选择两种方式：
> - 方式 A：在本机 WSL / Docker 中运行 Linux 版 Redis
> - 方式 B：下载社区维护的 Windows 版 Redis（学习/开发用途）

#### 方式 A：通过 WSL/Docker（推荐）

1. 启用 WSL 或安装 Docker Desktop（略）。  
2. 在 WSL/Docker 中运行：

```bash
docker run -d --name redis-teadrink -p 6379:6379 redis:6-alpine
```

3. 通过 Redis 客户端或 `redis-cli` 测试连接：

```bash
redis-cli -h 127.0.0.1 -p 6379 ping
```

若返回 `PONG` 即成功。

#### 方式 B：Windows 版 Redis（仅学习环境）

1. 搜索并下载 `Memurai` 或社区维护的 `Redis for Windows`。  
2. 安装并按软件指引启动服务，默认端口一般为 6379。  
3. 使用客户端测试 `ping`。

### 5.2 后端中 Redis 的用途（结合定时任务）

- 缓存高频访问数据：
  - 商品信息（如 `product:{id}`、商品列表缓存等）
  - 库存信息（`stock:{ingredientId}`）
  - 会员信息（`member:{id}`、`member:phone:{phone}`）
- 不使用 Canal，而是通过 **Spring 定时任务** 从 MySQL 补刷这些数据：
  - 例如每隔 5 分钟刷新商品与库存缓存；
  - 对于写操作，仍以 MySQL 为准，并可在写操作后同步更新 Redis（双层保证）。

具体定时任务逻辑会在后端代码中实现，例如：

- 使用 `@EnableScheduling` 开启定时任务；
- 编写 `@Scheduled(cron = \"0 */5 * * * ?\")` 方法，每 5 分钟全量/增量刷新。

## 六、Nginx 环境与部署

### 6.1 安装 Nginx（Windows）

1. 前往 Nginx 官网下载 Windows 版压缩包。  
2. 解压到如：`C:\nginx`。
3. 在 `C:\nginx` 下，通过命令行执行：

```bash
nginx.exe
```

4. 打开浏览器访问 `http://localhost`，若看到 Nginx 欢迎页，即启动成功。

### 6.2 Nginx 配置前后端

假设：

- 后端 Spring Boot 端口：`8080`
- 前端构建输出目录：`d:\teadrinkSystem\frontend\dist`

可在 `conf/nginx.conf` 中添加类似配置（示意，实际文件中可能需要合并）：

```nginx
server {
    listen       80;
    server_name  localhost;

    # 前端静态资源
    root  d:/teadrinkSystem/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理（假设接口前缀为 /api）
    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 静态上传资源（如商品图片）
    location /static/ {
        alias d:/teadrinkSystem/upload/;
    }
}
```

修改配置后可通过：

```bash
nginx -s reload
```

来重载配置。

## 七、后端 Spring Boot 基础配置思路

> 这里只说明环境相关关键配置项，具体 `application.yml` 会在后续后端工程中单独提供。

### 7.1 数据源与 Redis

在 `backend` 工程的 `application.yml` 中，需配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/teadrink_system?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8
    username: teadrink
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver

  redis:
    host: 127.0.0.1
    port: 6379
    # password: your_redis_password   # 如有密码则启用
```

### 7.2 定时任务开启

在 Spring Boot 启动类或配置类上添加：

```java
@EnableScheduling
@SpringBootApplication
public class TeadrinkApplication {
    public static void main(String[] args) {
        SpringApplication.run(TeadrinkApplication.class, args);
    }
}
```

然后在某个 Service 中编写同步任务（示意）：

```java
@Service
public class CacheSyncService {

    @Scheduled(cron = "0 */5 * * * ?")
    public void syncProductAndStockToRedis() {
        // 从 MySQL 查询商品、库存数据，写入 Redis
    }
}
```

## 八、推荐的搭建顺序（从零开始）

1. **安装 JDK + Maven**，验证 `java`、`javac`、`mvn` 命令可用。
2. **安装 Node.js**，验证 `node`、`npm` 命令可用。
3. **安装 MySQL**，创建 `teadrink_system` 数据库，后续执行 `db/` 中 SQL。
4. **安装 Redis**，确保本机 `6379` 端口服务正常。
5. **初始化后端工程**（放在 `backend/`，稍后会生成骨架与配置）。
6. **初始化前端工程**（放在 `frontend/`，使用 Vite + Vue3 + Element Plus）。
7. **安装并配置 Nginx**，验证能打开前端页面，并能代理到后端 `/api`。
8. 在后端实现 **定时任务 + Redis 缓存**，替代 Canal 的 binlog 监听。

完成以上步骤后，即可进入业务功能开发与联调阶段。
