# 联调测试：报表模块（销售日报）

本测试基于 **Nginx 同源联调**（前端静态 + `/api` 反代后端 8080）。

## 一、接口范围

- `GET /api/v_daily_sales?limit=30`

## 二、页面联调步骤

1. 打开 `http://127.0.0.1/html/Login.html` 登录进入主界面
2. 进入 **营业报表** 页面
3. 检查“销售日报”表格是否正常显示数据
4. 同时检查“商品销售排行”是否仍正常（回归）

## 三、Network 断言（F12）

`GET /api/v_daily_sales?limit=30` 返回：
- 外层：`{success,msg,data}`
- `data[]` 每条包含：
  - `sale_date`
  - `order_count`
  - `total_amount`
  - `discount_amount`
  - `real_income`
  - `cash_amount`
  - `mobile_amount`

前端会直接用这些字段渲染日报表格。

## 四、常见问题

- **暂无数据**：当天/近期没有已完成订单（`t_sale_order.status=1`）
- **500 报错**：确认数据库已执行 `db/views.sql`，存在 `v_daily_sales` 视图
- **401**：重新登录确保请求带 `X-Token`
