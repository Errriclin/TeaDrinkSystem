# 联调测试：会员充值模块

## 一、接口

- `POST /api/member/{id}/recharge`

请求体：

```json
{
  "amount": 100,
  "remark": "前端会员列表充值"
}
```

## 二、页面联调步骤

1. 登录进入系统，切换到“会员中心”
2. 在会员列表点击某条会员的“充值”
3. 输入金额（大于0）
4. 充值成功后列表自动刷新，会员余额增加

## 三、数据库验收点

- `t_member.balance` 增加对应金额
- `t_member_account_log` 新增一条：
  - `biz_type = RECHARGE`
  - `delta_balance > 0`
  - `balance_after` 为充值后余额
  - `remark` 记录充值备注

## 四、异常场景

- 金额 <= 0：前端拦截并提示
- 会员冻结：后端返回业务错误
- 会员不存在：后端返回业务错误
