# 联调测试：订单管理模块（列表/详情）

本测试基于 **Nginx 同源联调**（前端静态 + `/api` 反代后端 8080）。

## 一、接口范围

- `GET /api/sale_order/list?page=1&size=20`
- `GET /api/sale_order/{orderNo}/detail`

## 二、页面联调步骤

1. 打开 `http://127.0.0.1/html/Login.html` 登录进入主界面
2. 进入 **订单管理** 页面
3. 观察表格是否能加载出数据
4. 点击任意一行的 **“详情”** 按钮，弹窗应显示商品明细与金额汇总

## 三、Network 断言（F12）

### 1）列表接口返回结构

`GET /api/sale_order/list?page=1&size=20` 返回：

- 外层：`{success,msg,data}`
- `data` 内：`{total, list, page, size, pages}`
- `data.list[]` 每条包含（前端渲染所需）：
  - `order_no`
  - `created_at`
  - `member_name`
  - `items_count`
  - `total_amount`
  - `discount_amount`
  - `pay_amount`
  - `pay_type`
  - `cashier`
  - `status`

### 2）详情接口返回结构

`GET /api/sale_order/{orderNo}/detail` 返回：

- `order_no,status,created_at,total_amount,discount_amount,pay_amount,pay_type,cashier,member_name,member_level`
- `items` 数组，每项包含：
  - `product_name, unit_price, quantity, subtotal`

## 四、常见问题

- **列表为空**：先去 POS 下几单（`POST /api/sale_order`）再刷新订单管理页
- **详情报“订单不存在”**：检查前端传的 `orderNo` 是否正确（表格里 `order.order_no`）
- **401**：重新登录，让浏览器带上 `X-Token`
