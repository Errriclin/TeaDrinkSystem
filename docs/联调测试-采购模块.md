# 联调测试：采购模块（创建 / 列表 / 确认入库 / 详情）

本测试基于 **Nginx 同源联调**（前端静态 + `/api` 反代后端 8080）。

## 一、接口范围

- `GET /api/purchase_order/list`
- `POST /api/purchase_order`
- `POST /api/purchase_order/{id}/confirm_inbound`
- `GET /api/purchase_order/{id}/detail`

## 二、页面联调步骤

1. 打开 `http://127.0.0.1/html/Login.html` 登录进入主界面
2. 进入 **采购管理**
3. 点击“新建采购单”，填写供应商、原料、数量、单价并提交
4. 列表应新增采购单，状态为“待入库”
5. 点击“确认入库”
6. 切到 **库存管理**，验证原料库存增加，库存流水出现“采购入库”
7. 回到采购页点“详情”，弹窗（alert）应展示采购单与明细

## 三、Network 断言（F12）

- `POST /api/purchase_order` 返回：`{success:true,data:{id,order_no/orderNo...}}`
- `POST /api/purchase_order/{id}/confirm_inbound` 返回：`{success:true,...}`
- `GET /api/purchase_order/list` 返回列表字段包含：
  - `id,order_no,supplier,total_amount,status,operator,created_at`
- `GET /api/purchase_order/{id}/detail` 返回：
  - 基本信息：`order_no,supplier,total_amount,status,operator,created_at,inbound_at`
  - 明细：`items[]`（`material_name,quantity,unit_price,subtotal`）

## 四、常见问题

- **新增失败 401**：重新登录（采购接口需要 `X-Token`）
- **确认入库失败**：采购单不是“待入库”状态，或明细为空
- **库存没变化**：检查是否确实点击了“确认入库”并返回成功
