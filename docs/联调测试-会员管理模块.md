# 联调测试：会员管理模块（列表 / 新增 / 搜索）

本测试基于 **Nginx 同源联调**（前端静态 + `/api` 反代后端 8080）。

## 一、接口范围

- `GET /api/member/list`
- `POST /api/member`
- `GET /api/member/search?keyword=xxx`
- （POS 已有）`GET /api/member?phone=xxx`

## 二、页面联调步骤

1. 打开 `http://127.0.0.1/html/Login.html` 登录进入主界面
2. 进入 **会员管理** 页面
3. 验证列表是否加载成功
4. 点击“新增会员”，输入手机号/姓名/等级/初始余额，提交后列表应刷新并出现新数据
5. 在搜索框输入手机号或姓名关键字，列表应过滤（或触发后端搜索版本）

## 三、Network 断言（F12）

### 1）会员列表

`GET /api/member/list` 返回：
- `{success,msg,data}`
- `data[]` 每条包含：`id,phone,name,level,balance,points,total_consume,status,created_at`

### 2）新增会员

`POST /api/member` 请求体（前端提交）：
- `{phone,name,level,balance}`

返回：
- `{success:true,msg:"ok",data:{...member}}`

### 3）搜索会员

`GET /api/member/search?keyword=xxx` 返回：
- `{success,msg,data}`，`data` 为匹配列表

## 四、常见问题

- **401**：重新登录，确保请求头带 `X-Token`
- **新增失败**：看返回 `msg`（如手机号为空）
