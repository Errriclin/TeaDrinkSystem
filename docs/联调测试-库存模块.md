# 联调测试：库存模块（原料列表 + 库存流水）

本测试基于 **Nginx 同源联调**（前端静态 + `/api` 反代后端 8080）。

## 一、接口范围

- `GET /api/material/list`
- `GET /api/inventory_log/recent?limit=10`

## 二、页面联调步骤

1. 打开 `http://127.0.0.1/html/Login.html` 登录进入主界面
2. 进入 **库存管理** 页面
3. 观察：
   - 原料表格能加载
   - 低库存原料行会高亮（`is_low = 1`）
   - 点击“仅看预警/低库存”（若有该按钮）能筛选
   - 右侧/下方库存流水能加载出“销售出库”等记录

## 三、Network 断言（F12）

### 1）原料列表

`GET /api/material/list` 返回：

- 外层：`{success,msg,data}`
- `data[]` 每条包含：
  - `id,name,unit,stock_quantity,safety_stock,status,updated_at,is_low`

其中 `is_low` 为 0/1，前端会用它来：
- 行高亮
- 统计工作台低库存数量

### 2）库存流水

`GET /api/inventory_log/recent?limit=10` 返回：

- 外层：`{success,msg,data}`
- `data[]` 每条包含：
  - `material_name`
  - `quantity`（正入库、负出库；前端用它判断红/绿）
  - `after_stock`
  - `type_name`
  - `created_at`

## 四、常见问题

- **库存流水为空**：先去 POS 下几单（会写入“销售出库”库存流水）
- **低库存不高亮**：确认 `t_material.stock_quantity < safety_stock` 且 `status=1`
- **401**：重新登录确保请求头带 `X-Token`
