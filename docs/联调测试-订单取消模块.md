# 联调测试：订单取消模块（回补库存）

本测试基于 **Nginx 同源联调**（前端静态 + `/api` 反代后端 8080）。

## 一、接口范围

- `POST /api/sale_order/{orderNo}/cancel`

## 二、页面联调步骤

1. 登录后进入 **订单管理**
2. 选择状态为“已完成”的订单，点击“取消”
3. 确认弹窗后，订单状态应变为“已取消”
4. 切换到 **库存管理**，查看原料库存是否回补
5. 查看库存流水是否新增“订单取消回库”
6. 切换到 **工作台**，确认订单统计变化

## 三、Network 断言（F12）

- 请求：`POST /api/sale_order/{orderNo}/cancel`
- 返回：`{success:true,msg:"ok",data:null}`
- 若重复取消同一订单：返回业务错误（例如“订单已取消，无需重复操作”）

## 四、事务断言

取消订单应一次性完成以下动作（不可部分成功）：
- `t_sale_order.status` 从 `1` -> `0`
- 按配方将原料库存回补
- `t_inventory_log` 写入回库流水：
  - `biz_type = SALE_CANCEL_IN`
  - `type_name = 订单取消回库`
