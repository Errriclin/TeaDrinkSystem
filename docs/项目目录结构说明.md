# 项目目录结构说明

| 文档信息 | |
|---------|--|
| **目的** | 约定前后端、配置、脚本在项目中的**固定位置**，便于手动查找与修改 |
| **版本** | V1.0 |

---

## 一、仓库根目录总览

```
teadrinkSystem/
├── backend/                 # 后端 Java（Spring Boot），严格三层架构
├── front-end/               # 前端静态页面与脚本（当前技术栈：HTML + JS + CSS）
├── db/                      # MySQL：建表、视图、触发器、初始化数据等 SQL
├── deploy/                  # Nginx 等部署配置示例（不与业务代码混放）
├── docs/                    # 需求、计划、规范、说明文档
├── logs/                    # 测试记录、自动化测试结果日志（见 docs/测试日志说明.md）
└── previous/                # 历史/旧代码归档（非本期主工程，勿与 backend 混淆）
```

**原则**：业务代码只放在 **`backend/`** 与 **`front-end/`**；文档只放在 **`docs/`**；数据库脚本只放在 **`db/`**。

---

## 二、后端 `backend/`（Java / IDEA）

后端工程根目录为 **`backend/`**，在 IDEA 中以 **「打开目录」或「导入 Maven 项目」** 指向该文件夹。

建议包根：`com.teadrink`（若你改名，全项目统一修改）。

```
backend/
├── pom.xml
├── README.md                         # 可选：如何启动、端口说明
└── src/
    ├── main/
    │   ├── java/com/teadrink/
    │   │   ├── TeadrinkApplication.java    # 启动类（创建工程后生成）
    │   │   ├── controller/                 # 控制层：仅处理 HTTP 与入参
    │   │   ├── service/                    # 业务层接口
    │   │   │   └── impl/                   # 业务层实现（事务写在这里）
    │   │   ├── mapper/                     # 持久层：MyBatis-Plus Mapper
    │   │   ├── entity/                     # 与表对应的实体（或 domain）
    │   │   ├── dto/                        # 请求/响应、与前端 JSON 对齐的对象
    │   │   ├── config/                     # Spring、拦截器、跨域等配置
    │   │   └── common/                     # 统一返回体、异常、常量等
    │   └── resources/
    │       ├── application.yml             # 主配置（或 application-dev.yml）
    │       └── mapper/                     # 可选：自定义 XML Mapper
    └── test/
        └── java/com/teadrink/              # 单元测试 / 集成测试
```

**禁止**：在 `controller` 里写复杂业务；在 `mapper` 里写业务流程（见 `docs/初期目标与开发规范.md`）。

---

## 三、前端 `front-end/`（页面与静态资源）

当前前端为**多页 + 原生 JS**，统一放在 **`front-end/`**，按类型分子目录，**不要**把 `.html`、`.js`、`.css` 散落在仓库根目录。

```
front-end/
├── html/                   # 页面入口
│   ├── Login.html
│   ├── Register.html
│   └── Mainwindow.html
├── js/                     # 脚本（接口调用、页面逻辑）
│   └── index.js
├── css/                    # 样式
│   └── style.css
└── assets/                 # 可选：图片、字体、本地图标等（无则可为空）
```

- 若后续增加页面：新建文件放入对应 **`html/` / `js/` / `css/`**（或 `assets/`）。
- Nginx 静态根目录应指向 **`front-end/`**（或指向内含 html/js/css 的构建输出目录），保证路径与页面里引用的 `/js/`、`/css/` 一致。

---

## 四、数据库 `db/`

```
db/
├── ER设计与表结构概要.md   # 逻辑设计（精简版表清单）
├── schema.sql              # 建表（后续补充）
├── views.sql               # 视图（后续补充）
├── triggers.sql            # 触发器（后续补充）
└── init_data.sql           # 初始化数据（后续补充）
```

使用 **DataGrip** 打开并执行上述 SQL，与后端连接**同一 MySQL 库**。

---

## 五、部署配置 `deploy/`

```
deploy/
├── nginx/
│   └── teadrink.conf.example    # Nginx 反向代理 + 静态文件示例（后续补充）
└── README.md                    # 可选：部署步骤摘要
```

**不放** Java 源码或前端业务代码，仅放与环境、反向代理相关的配置示例。

## 六、文档与日志

| 目录 | 用途 |
|------|------|
| `docs/` | 需求、计划、环境、测试说明、**初期目标与开发规范**、**本目录结构说明** |
| `logs/` | `测试记录.md`、`test-results.log` 等（随测试追加） |

---

## 七、与旧目录 `previous/` 的关系

`previous/` 内为早期或其它技术栈残留文件，**本期联调与提交作业以 `backend/` + `front-end/` + `db/` 为准**。避免在 `previous/` 中继续堆叠新代码。

---

## 八、变更约定

若新增一类文件（例如 `front-end/vendor/` 存放第三方库），在本文件追加一节说明用途与命名规则，保持**分类清晰、路径可预期**。
