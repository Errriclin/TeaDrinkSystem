# 前后端联调测试说明（Nginx 同源）

本说明用于在 **Windows** 上完成本项目的前后端联调：由 Nginx 提供前端静态页面，并将 `/api` 反向代理到后端 SpringBoot（8080），从而实现 **同源** 调用（避免跨域、避免前端写死 IP）。

## 一、已完成的联调改动（你无需再改）

- **前端**：`frontend/js/index.js`
  - `API_BASE_URL` 已改为 `''`，因此所有请求会变成同源的 `/api/**`
  - token 丢失/401 时跳转地址已统一为 `/html/Login.html`
- **登录页**：`frontend/html/Login.html`
  - 登录请求已改为同源 `/api/login`
  - “立即注册”链接改为 `/html/Register.html`
- **Nginx 配置**：`nginx-tea/conf/nginx.conf`
  - 静态目录：`d:/teadrinkSystem/frontend`
  - `/api/` 反代：`http://127.0.0.1:8080`

## 二、启动顺序（必须按这个来）

### 1）启动后端（SpringBoot）

确保后端在 8080 端口启动成功。

自测：
- 在浏览器访问 `http://127.0.0.1:8080/api/health` 能返回 `ok`

### 2）启动 Nginx（静态 + 反代）

打开 PowerShell，进入项目的 Nginx 目录：

```powershell
cd d:\teadrinkSystem\nginx-tea
.\nginx.exe
```

检查 Nginx 是否启动成功：
- 在浏览器打开 `http://127.0.0.1/html/Login.html`

常用命令：

```powershell
cd d:\teadrinkSystem\nginx-tea
.\nginx.exe -s reload
.\nginx.exe -s stop
```

> 如果 80 端口被占用或没有权限，把 `nginx-tea/conf/nginx.conf` 中的 `listen 80;` 改成 `listen 8081;`，然后用 `http://127.0.0.1:8081/html/Login.html` 访问即可。

## 三、联调测试步骤（按页面操作）

### 1）登录

访问：
- `http://127.0.0.1/html/Login.html`

输入你初始化数据中的账号/密码登录（例如你之前用于测试的那组）。

预期：
- 登录成功后会跳转到 `/html/Mainwindow.html?token=...`
- 浏览器本地存储会写入 token（用于后续请求）

### 2）打开工作台并确认 4 个接口都成功

进入主界面后，切换到“工作台”（或默认就是工作台）。

按 `F12` 打开开发者工具 → `Network`，确认以下请求均返回 200：

- `GET /api/dashboard/summary`
  - 返回体是一个对象，直接包含：`todayRevenue/todayOrders/...`
- `GET /api/v_product_rank?limit=5`
  - 返回体是 `{success,msg,data}`
- `GET /api/sale_order/recent?limit=5`
  - 返回体是 `{success,msg,data}`
- `GET /api/reports/revenue-trend?days=7`
  - 返回体是 `{success,msg,data}`，其中 `data.labels` 与 `data.data` 数组长度一致

预期页面表现：
- 顶部统计卡片有数字（或为 0，但页面不报错）
- 热销商品列表能渲染（无数据则显示“暂无数据”）
- 最近订单表格能渲染（无数据则显示“暂无订单”）
- 营收图表能渲染（无数据也应该能渲染出 0 的柱子）

## 四、常见问题排查（最常见三类）

- **工作台请求 401**
  - 说明没带 `X-Token` 或 token 失效
  - 解决：回到登录页重新登录；确认请求头里有 `X-Token`

- **工作台数据都是 0 / 空列表**
  - 多半是数据库里没有“今天”的销售订单
  - 解决：执行 `db/init_data.sql`，或后续完成 `POST /api/sale_order` 后下几单再看

- **打开页面 404（路径不对）**
  - 正确入口是 `/html/Login.html`（不是 `/Login.html`）
